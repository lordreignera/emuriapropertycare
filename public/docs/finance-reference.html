Payment Plans Explained

💳 Full Payment

Client pays 100% of the ARP upfront in two steps.

— Step 1: 50% deposit at agreement signing
— Step 2: Remaining 50% before first work visit (or immediately after signing, client's choice)
— No ongoing invoices after full payment received

📋 Per Visit

Client pays for each work visit as it is completed.

— Step 1: 50% deposit at signing (covers the early visits)
— Subsequent invoices: Generated after each completed visit
— Amount per visit = TRC ÷ total number of planned visits
— Invoice is emailed to client after the visit log is saved

🗓️ Installment

Remaining balance is split into agreed regular installments.

— Step 1: 50% deposit at signing
— Remaining 50% divided into monthly or milestone-based installments
— Installment schedule is agreed during the Pre-Sign Setup stage
— Finance Officer generates each installment invoice on schedule
The payment plan is agreed during contract setup. To change a plan mid-project, a Change Order signed by both parties is required.

Viewing Invoices

Go to Billing & Finance → Unpaid Invoices to see only outstanding payments, or All Invoices for the complete history. You can filter by client, project, or date range.

Creating a Manual Invoice

  • 1

    Go to Billing & Finance → All Invoices

    Click Create Invoice.

  • 2

    Select the Client & Project

    The invoice will be linked to the specific project so the client can see it from their dashboard.

  • 3

    Set Amount, Description & Due Date

    Enter the invoice amount, a clear description (e.g. "Installment 2 of 3"), and the payment due date.

  • 4

    Save & Notify

    Save the invoice. The client receives a notification and can see the "Pay Now" button appear on their project card and in Billing → Invoices.

Stripe Integration & Reconciliation

EMURIA uses Stripe for all client payment processing. The system listens to Stripe webhook events to automatically update invoice and payment status.

How Payment Updates Work

// Automatic flow (no Finance Officer action needed) Client clicks "Pay Now" on their dashboard → Stripe Checkout page opens → Client enters card details → Stripe processes payment → Stripe sends webhook event to EMURIA → Invoice status updated to Paid automatically → Client receives email confirmation → Finance Officer sees updated status in Billing

Stripe Dashboard Reconciliation

Stripe EventWhat It Means in EMURIA
checkout.session.completedPayment successful — invoice marked Paid, project advances
payment_intent.payment_failedCard declined — invoice remains Pending, client notified
charge.refundedRefund issued — Finance Officer should void the invoice and note reason
invoice.paidSubscription payment received (for subscription clients)
If a payment shows as successful in Stripe but not updated in EMURIA: Check the webhook logs in the Stripe dashboard for delivery failures. Contact your system administrator to manually trigger a webhook resync.

Issuing a Refund

Refunds must be processed directly in the Stripe Dashboard (stripe.com). After issuing the refund in Stripe:

  1. Go to Billing & Finance → All Invoices in EMURIA
  2. Find the invoice — click Edit → change status to Void
  3. Add a note explaining the refund reason and date
  4. Notify the client through the Help & Support system

Common Finance Scenarios

ScenarioAction
Client has paid but system still shows "Unpaid"Check Stripe webhook delivery in Stripe Dashboard. If confirmed paid in Stripe, manually mark the invoice as Paid in EMURIA and add a note. Contact system admin if webhook is failing.
Client paid the wrong amountVoid the original invoice. Create a new invoice for the correct amount. If client overpaid, issue a partial refund via Stripe for the difference.
Project cancelled after 50% deposit paidReview cancellation policy. If refund is due, issue via Stripe. Void relevant invoices. Close the project in the system with a cancellation note.
Client disputes an invoice amountDo not void immediately. Review the ARP calculation in the inspection pricing dashboard. If an error is found, void and reissue. If correct, provide the client with a pricing breakdown from the inspection report.
Need to send a payment reminderGo to the overdue invoice in EMURIA → use the "Send Reminder" button, or contact the client through Help & Support messaging.
Client wants to change their payment planRequires a Change Order signed by both parties. After approval, update the invoice schedule accordingly.
