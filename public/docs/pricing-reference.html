Step 1 — Base Deployment Cost (BDC)

BDC covers the cost of deploying the Emuria team to the property — travel distance, travel time, and annual frequency.

// Per-Visit Calculation BDC_per_visit = (distance_km × rate_per_km) + (minutes × rate_per_minute) // Example: 40 km, 30 min travel, rate=$1.50/km, $1.65/min BDC_per_visit = (40 × 1.50) + (30 × 1.65) = 60.00 + 49.50 = $109.50 // Annual BDC BDC_annual = BDC_per_visit × visits_per_year = 109.50 × 24 = $2,628.00/year
Input FieldDescriptionConfigurable?
bdc_distance_kmOne-way travel distance to the property in kmPer inspection (inspector enters)
bdc_time_minutesOne-way travel time in minutesPer inspection (inspector enters)
bdc_visits_per_yearNumber of annual visits — auto-derived from total labour hours ÷ 8Auto-calculated
rate_per_kmCost per km — default $1.50BDC Calibration Engine settings
rate_per_minuteCost per travel minute — default $1.65BDC Calibration Engine settings
Admin action: Go to System Settings → BDC Calibration Engine to update rate_per_km and rate_per_minute for all future inspections.
🔨

Step 3B — FRLC & FMC (From Findings)

Each finding entered by the inspector has a labour hours estimate and a material cost. These are summed across all approved findings.

// Field Resource Labour Cost (per finding) FRLC_finding = phar_labour_hours × labour_rate_per_hour // Total FRLC (all approved findings) FRLC_annual = SUM(FRLC_finding) for all approved findings // Field Material Cost FMC_annual = SUM(material_cost) for all approved findings // Example: 15 findings, avg 5 hrs each @ $125.40/hr, avg $72 materials FRLC_annual = 15 × 5 × 125.40 = $9,405.00 FMC_annual = 15 × 72 = $1,080.00
Labour rates per hour are configured in System Settings → FMC Material Settings. Material costs are set per-finding in the Findings Template Settings.
➕

Step 4 — TRC (Total Remediation Cost) — Merge Bridge

// Total Remediation Cost TRC_annual = BDC_annual + FRLC_annual + FMC_annual = 2,628 + 9,405 + 1,080 = $13,113.00/year TRC_monthly = TRC_annual / 12 = $1,092.75/month TRC_per_visit = TRC_annual / visits_per_year // ARP = TRC (the client-facing Annual Recurring Price) ARP = TRC_annual = $13,113.00
ARP is the final project price. The client sees this as their total project cost. The 50% deposit = ARP ÷ 2.
🏷️

Step 5 — Service Tier Assignment (Dual-Gate)

The service tier is determined by two gates — whichever gate produces the higher tier wins.

// Gate 1: Condition Score (from CPI assessment) // Score 0–100 (lower = worse condition) if condition_score <= 30 → Tier_Score = "Critical Care" if condition_score <= 50 → Tier_Score = "White-Glove" if condition_score <= 70 → Tier_Score = "Premium" else → Tier_Score = "Essentials" // Gate 2: Cost Pressure (ARP vs base package prices) if ARP_monthly >= 1800 → Tier_ARP = "Critical Care" if ARP_monthly >= 1200 → Tier_ARP = "White-Glove" if ARP_monthly >= 703 → Tier_ARP = "Premium" else → Tier_ARP = "Essentials" // Final Tier = highest of the two gates Tier_Final = MAX(Tier_Score, Tier_ARP)
Essentials   Condition score 71–100  |  ARP < $703/month  |  Multiplier: 1.00×
Premium   Condition score 51–70  |  ARP $703–$1,199/month  |  Multiplier: 1.15×
White-Glove   Condition score 31–50  |  ARP $1,200–$1,799/month  |  Multiplier: 1.35×
Critical Care   Condition score 0–30  |  ARP ≥ $1,800/month  |  Multiplier: 1.55×
✖️

Step 6 — Multiplier Application (ARP Equivalent Final)

// Apply tier multiplier to arrive at the final ARP equivalent ARP_Equivalent_Final = ARP_monthly × Multiplier_Final // Example: White-Glove tier, ARP $1,092.75 ARP_Equivalent_Final = 1,092.75 × 1.35 = $1,475.21/month // Floor rule: never below base package price for the tier Scientific_Final = MAX(ARP_Equivalent_Final, base_package_price)
The multiplier reflects the service complexity and commitment level of that tier. Clients at Critical Care tier receive the most intensive service regime and the highest SLA.
